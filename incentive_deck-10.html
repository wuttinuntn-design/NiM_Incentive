วิธีคิด Incentive — 3 Model

เลือกวิธีคิดให้เหมาะกับความพร้อม

แต่ละ Model ให้ผลต่างกัน — มีเส้นทางที่แนะนำ
MODEL 01
เริ่มได้เลย
งบคงที่ตาม Grade
งบ Fix/เดือน → แจกตาม Score: A=100% · B=70% · C=0% · D,F=หัก
ไม่ต้องรอ Baseline · คุมงบได้แน่นอน · งบไม่ผูกผลประหยัดจริง
MODEL 02
ระยะยาว
Gain Sharing
ส่วนต่าง cost (จริง vs Target) × รายได้รวม × 50% = งบ Incentive
Win-Win ชัด · ยุติธรรม ไม่มีความเสี่ยงจ่ายเกิน · ต้องมี Baseline แม่นยำก่อน
MODEL 03
ระยะยาวมาก
Pay per Performance
ส่งสำเร็จ >97% → X บาท/บิล · ต้นทุน <20% → Y บาท/เที่ยว
เชื่อมพฤติกรรมกับเงินโดยตรงที่สุด · ซับซ้อน ต้องออกแบบ Rate รอบคอบมาก
เทียบ 3 Model อย่างย่อ
ความเร็วเริ่มงาน: Model 1 เร็วสุด
ความยุติธรรม: Model 2 สูงสุด
แรงจูงใจตรงจุด: Model 3 สูงสุด

ข้อมูลย้อนหลังที่ต้องไปดึงจากระบบจริง

แยกตามคนรับผิดชอบ — ใช้คำนวณ Baseline ก่อนล็อก Threshold จริง
ทีมการเงิน/บัญชี — ย้อนหลัง 3-6 เดือน
  • ยอดขาย/รายได้ ย้อนหลังรายเดือน แยกรายสาขา (AB + CC)
  • ต้นทุนค่าจ้างรถ ย้อนหลังรายสาขา (รถบริษัท vs รถร่วม)
  • เงินเดือน/ค่าจ้างปัจจุบันแต่ละตำแหน่ง (เช็คผลกระทบฐานเดิม)
ทีมปฏิบัติการ — ย้อนหลัง 3-6 เดือน
  • จำนวนบิลเข้า-บิลออก ย้อนหลังรายสาขา
  • % จัดส่งสำเร็จ และ % เสียหาย/สูญหาย ย้อนหลังรายสาขา
  • Cut-off time ขนย้ายจริงของแต่ละสาขา (ไม่เท่ากัน)
ทีม HR / ยานยนต์ — ข้อมูลปัจจุบัน
  • จำนวนรถกระจายจริงต่อสาขา แยก AB/CC (ไม่ใช่ตัวสมมติ)
  • รายชื่อ พขร. แยกพนักงาน vs ผู้ประกอบการ ต่อสาขา
  • จำนวนพนักงานแต่ละสาขา แยกตำแหน่ง/Level (LV4-8)
ทีม IT/ระบบ — สำรวจ Data Source
  • ระบบที่เก็บข้อมูลแต่ละตัวอยู่ตอนนี้ (SAP/Excel/Manual)
  • ความถี่ที่ดึงข้อมูลอัตโนมัติได้ vs ต้องกรอกมือ
  • เจ้าของข้อมูลแต่ละระบบ (Contact point)
แนะนำ: ย้อนหลังขั้นต่ำ 3 เดือนพอเริ่ม Sim ได้ แต่ 6 เดือนจะเห็น pattern ตามฤดูกาล/เทศกาลชัดกว่า เหมาะกับการตั้ง Threshold ที่แม่นยำ
